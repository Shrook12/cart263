PROJECT ONE RESPONSE


The first project I am going to discuss is Worm Fishing Survival by Ben. The experience of the game itself is really amazing. I enjoyed the animations of the background and of everything else, and the simplistic illustrations.  The background is constantly moving with the other small details in  give a real sense of being underwater. The attack animation, with the bubble when clicking the mouse, is really a nice touch. I also like how the worm stays centered, not able to moving freely,just like in real life, and it makes the game simpler and easier to understand because you don’t need to click a lot of buttons at the same time. The code is well structured and clearly organized with small shorts comments for each small code section to explain what each piece of code is doing without writting too much. One thing I was surprised by was that I discovered that all the illustrations were coded and did not use images. I can only imagine how time-consuming that must have been. Creating the whole visual of a project by programming can sometimes be really hard, so building a complete game with such a good illustration style without using a single image is truly amazing.

The third project is an interactive project where the fish follow the mouse, done by Philippe. The interaction is really cool, and it feels smooth. I like how they are all move freely everywhere in the canvas, and when you start moving your mouse, they all come together moving to follow the mouse. The small details of the animation is really amazing because it’s small details like how the fish turn their direction when they bound to the edge or touch another fish gives a very realisitic fish movement feel. Also, the way the fish wobble slightly from time to time when they follow the mouse adds a nice touch to the animation. For the code, I like that it uses a constructor. I did not see many other projects that use it. Having a constructor with separated files makes the code more organized and easier to understand. I also like how he wrote the source he used at the top of each code.
